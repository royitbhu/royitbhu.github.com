1. simple steps on how to change the password using single user mode
  1. Reboot the Ubuntu
  2. When grub is loading,press 'Esc'
  3. Choose 'Ubuntu kernel(recovery mode)'
  4. Press 'e' to edit the kernel parameter
  5. Append 'single init=/bin/bash' to the kernel parameter
  6. Press 'b' to boot from that particular kernel

    7. You will enter single user mode . Your hard drive will be in read-only mode. Remount it in read-write mode using following command.

  7. "$ mount -o remount,rw /dev/sda1 "(drive name in which ubuntu is installed)
  8. Change your passwd : "$ passwd user"
  9. Reboot your machine and enter newly changed password
  10. Hurrey !!! Password has been changed

4. Change Default Boot Options
    When we install Ubuntu is set to be the default operating system to boot up .This order can be changed easily with startup manager.

  1. Open Terminal
  2. Install startup manager using following command : sudo apt-get install startupmanager
  3. Go to : System > Administration > StartUp-Manager
  4. Select the default operating system from the pull-down menu, click "Close" to perform post-configuration tasks
5. Auto Shutdown the System
    To schedule a time for the system to shut down type the following command.

  1. "sudo shutdown -h hh:mm" (here hh:mm = 22:10hr)
  2. "sudo shutdown -h +m" (here +m = 50 minutes)
6. Ubuntu command line (terminal) using tips
    keyboard shortcuts you can use within terminal.

  1. "Ctrl-a": Move to the start of the line
  2. "Ctrl-e": Move to the end of the line
  3. "!! ": Execute last command in history
  4. "!abc": Execute last command in history beginning with abc
  5. "Ctrl-r": Incremental reverse search of history
  6. "^abc^xyz": Replace first occurrence of abc with xyz in last command and execute it
  7. "Ctrl-l": Clear the screen leaving the current line at the top of the screen
7. Useful Git commands
  1. "git init": initializes a directory as a Git repository
  2. "git add": adds file contents to the staging area
  3. "git status": view the status of your files in the working directory and staging area (use "git status -s" for short output)
  4. "git diff": shows diff of what is staged and what is modified but unstaged (use -m for commit message)
  5. "git commit": records a snapshot of the staging area
  6. "git commit -a": if you deleted any files from local repo then delete those files from remote github repo
  7. "git remote": list repository aliases
  8. "git remote add [alias] [url]": add alias for a repo
  9. "git remote rm [alias]": remove alias from current repo
